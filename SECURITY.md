# Security Policy

## Reporting a vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Instead, report the issue privately through GitHub's private vulnerability
reporting flow if it is enabled for the repository. If that is not available,
contact the maintainers through a private channel and include:

- A clear description of the issue
- Steps to reproduce or a proof of concept
- The affected files, modules, or commands
- Any suggested mitigation if you already have one

You should receive an initial response within 5 business days.

## Disclosure expectations

- Give maintainers reasonable time to investigate and remediate.
- Avoid sharing exploit details publicly before a fix or mitigation is available.
- Keep reports factual and scoped to the observed impact.

## Supported versions

At the moment, only the latest development line is considered supported.

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |
| < 0.1.0 | No |
